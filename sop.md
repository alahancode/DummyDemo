### 2.2 本项目输入

1. Demo 名称
    OSINT Agent Console / AI-assisted Threat Investigation Demo

2. 一句话描述
    我要做一个 OSINT Agent 的高保真前端 demo，用来展示 AI 如何辅助安全分析师从一个 IOC 输入出发，完成多源威胁情报查询、证据链整理、风险判断和处置建议生成。

3. 目标观众
    这个 demo 主要给以下人看：
    - 面试官：用于展示我具备快速理解需求、设计 demo、使用 AI coding 交付原型的能力。
    - 导师 / 项目评审者：用于快速理解 OSINT Agent 项目的目标、流程和阶段性成果。
    - 解决方案客户 / 非安全工程背景受众：用于说明 AI Agent 在威胁情报分析中的业务价值。
    观众背景可能包括：
    - 有一定技术背景，但不一定熟悉 OSINT 或安全分析细节。
    - 关心 demo 是否能清楚表达业务价值、技术路径和后续落地可能性。
    - 不会逐行看代码，更关注 3 分钟内能否看懂这个系统在做什么。

4. 用户问题
    目标用户现在遇到的问题：
    1. 安全分析师拿到一个 IOC，例如 IP 或 domain 后，需要手动查询多个 OSINT 来源，流程分散、耗时。
    2. 不同情报源返回的信息格式不统一，难以快速形成可信证据链。
    3. 单个 threat score 或单条情报不能直接支持处置决策，需要解释“为什么可疑”。
    4. 非安全背景的评审者很难通过后端代码理解系统价值，需要一个可视化 demo 来对齐需求。
    5. 在 PoC 阶段，需要快速展示核心流程，而不是一开始就做完整生产系统。

5. 核心演示场景
    用户从输入一个 IOC 开始，例如：185.220.101.47
    系统展示一个模拟的 OSINT investigation 流程：
    用户输入 IOC。
    点击 Run Investigation。
    系统开始展示 Agent execution log。
    多个 OSINT source card 进入 querying / done / flagged 状态。
    页面生成调查报告，包括：risk level
    confidence
    threat summary
    evidence chain
    recommended actions
    用户切换到 Agent Lab / Source Inspector，查看每个 source 的 raw result、reasoning chain 和建议动作。
    用户可以通过 chat 区域追问系统，例如 blocking、false positive、confidence、next steps。
    最终结果是：观众能看到一个从 IOC 到证据链报告的完整闭环。

6. 必须展示的能力或画面
    我希望观众能直观看到：这个系统不是一个普通聊天机器人，而是围绕一个安全调查任务在工作。
    我希望 demo 能表现出几个感觉：
    1. 用户输入一个可疑 IP 或 domain 后，系统会开始调查，而不是直接给一句结论。
    2. 调查过程中能看到“系统正在查不同来源”的过程感。
    3. 最终结果不是一段大模型生成的空话，而是有证据、有风险判断、有推荐动作。
    4. 观众能理解：AI 在这里的价值是帮助整理证据、解释原因、辅助决策。
    5. 如果可以，希望有一个地方能展示每个情报来源查到了什么，但具体页面形式可以由 AI 帮我设计。

7. 数据与能力边界
    这个 demo 主要是为了展示流程和方案价值，不是要证明已经完成生产级系统。
    目前可以接受使用 mock 数据来模拟 OSINT 查询结果，比如某个 IP 在多个情报源里的返回情况。但我希望边界说清楚：这些数据是为了演示调查流程，不代表 HTML 页面真实调用了外部安全 API。
    如果涉及 LLM，我希望它主要扮演“解释和总结”的角色，比如把证据整理成风险说明、回答追问，而不是让它凭空编造威胁情报。
    后续如果接真实后端，可以把 mock 数据替换成 FastAPI / OSINT backend 返回的 investigation result。

8. 视觉方向
    我希望视觉上让人感觉这是一个专业的安全分析工具，而不是普通网页或者营销页。

    整体感觉可以偏：
    - 深色
    - 控制台
    - 安全分析
    - 数据密集
    - 专业、克制、有技术感
    我不确定具体布局应该怎么设计，可以让 AI 根据这个方向提出页面结构和视觉方案。
    不希望它看起来像 SaaS 官网、landing page、模板站，也不希望一上来就是大标题和宣传文案。它应该更像一个分析师正在使用的工作台。  

9. 技术约束
  交付形式：
  第一阶段交付：单文件 HTML
  内联 CSS
  内联 JavaScript
  可直接用浏览器打开
  不需要构建工具
  不需要后端服务

10. 成功标准
    这个 demo 成功的标准：
    打开单个 HTML 文件即可演示。
    观众 3 分钟内能理解系统价值。
    页面第一眼像专业安全分析工具。
    点击 Run Investigation 后，有完整的动态调查流程。
    source cards 状态会变化。
    execution log 会逐步出现。
    最终 report 包含 risk、confidence、summary、evidence、actions。
    Reset 可以恢复初始状态。
    Agent Lab 未运行前有明确提示。
    Agent Lab 运行后能查看 source detail。
    Chat 能展示“可解释 AI”的感觉，但不假装真实 LLM。
    所有 mock / demo-only 能力边界清楚。
    适合用于面试展示“我能快速理解需求并用 AI coding 交付 demo”。

11. 当前项目背景补充
     项目真实背景：
     项目名：OSINT Agent
     项目目标：输入一个 IP / domain 等 indicator，输出多源 OSINT evidence report。
     当前系统方向：Agentic threat intelligence analysis system。
     后端方向：FastAPI + OSINT fetchers + storage + agent orchestration。
     Agent 架构方向：Orchestrator / Executor / Reasoning。
     前端方向：展示 IOC investigation、evidence、logs、report、source detail。
     当前 HTML demo 定位：高保真 presentation shell，不代表真实后端 contract。
     当前求职定位：展示快速方案理解、demo 构建、PoC 边界控制、AI coding workflow。
